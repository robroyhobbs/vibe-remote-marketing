How to use communities
如何使用社区

Ask for feedback first. Distribution second.

先要反馈，再谈分发。

In the first month, the best post is not “please download my app.” It is “I built this for a problem I personally hated — does this feel useful or risky to you?”

在第一个月里，最好的帖子不是“请下载我的 App”，而是“我因为讨厌某个问题做了这个工具——你觉得它有用吗？哪里会让你犹豫？”

Who to DM first

  • Developer friends who travel or work away from their desk

  • Mac power users who already use Tailscale/SSH
  • Technical founders juggling multiple repos

DM template
私信模板

Short and honest beats polished

短、真诚，比精致更有效

Hey — I built a native iPhone remote for Claude Code / Codex because my own SSH-from-phone setup felt awful. It’s early, but working. If I sent you the link, would you try it and tell me what feels confusing or sketchy? I’m optimizing for honest feedback, not compliments.
