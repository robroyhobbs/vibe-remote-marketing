Activation prompts
激活提示词

The best first 3 prompts

最佳前 3 条提示词

For activation, these matter more than the full library. They get the user to the first “oh wow” moment fastest.

从激活角度看，这 3 条比完整库更重要，因为它们最容易把用户带到第一个 “oh wow” 时刻。

1. Explain this repo

1. 解释这个仓库

read the README and the last 5 git commits, then explain what this project does and what changed recently

2. Run and fix tests

2. 跑测试并修复

run the test suite, show me failures, then fix them one by one

3. Ship a hotfix

3. 发布热修复

create a branch fix/[issue], fix the bug, run tests, commit, push, and open a PR
Prompt library
提示词库

First 10 prompts

前 10 条提示词

“What did the AI build last night?”

“昨晚 AI 构建了什么？”

read git log --since='yesterday' and summarize what changed in plain english

“Something broke in prod.”

“线上出问题了。”

check the last 50 lines of logs in ./logs/error.log and tell me what is failing

“Remind me what this project does.”

“提醒我这个项目是干嘛的。”

read the README and the last 5 git commits. give me a 5-sentence summary of what this project does and where it's headed

“Build the feature.”

“把这个功能做出来。”

add [feature] to [module]. write tests first, then implement. commit when tests pass

“Review the PR before I merge.”

“合并前先 review 一下。”

show me the diff for the last PR on this branch and tell me if anything looks risky

“Is the server healthy?”

“服务还健康吗？”

check if the dev server is running. if not, start it and confirm it's up on port 3000

“Clean this up.”

“清理这段代码。”

look at [file], refactor for readability and consistency with the rest of the codebase, then show me what changed

“End of day wrap-up.”

“每天收尾总结。”

summarize what was built today, what tests are passing, what's still open, and write a one-paragraph standup update I can paste into Slack

“Start the project for me.”

“帮我先把项目跑起来。”

inspect the repo, install dependencies if needed, start the app, and tell me how to access it

“Prep for a demo.”

“帮我准备 demo。”

check that the app builds, the server runs, and summarize anything risky before my demo
From prompts to proof
从提示词到证据

How prompts become marketing assets

如何把提示词变成营销资产

  • Pick one real prompt and record the whole flow.
  • Turn the output into a screenshot carousel.
  • Write the story as: context → prompt → result → why it mattered.
  • Reuse the same proof across X, HN comments, docs, and Product Hunt.
  • 选择一个真实提示词并完整录下过程。
  • 把输出结果变成截图轮播。
  • 按“背景 → 提示词 → 结果 → 为什么重要”写成故事。
  • 同一份证据可以复用到 X、HN 评论、文档和 Product Hunt。
Best content formula
最佳内容公式

“Here’s the exact prompt I used from my phone.”

“这是我在手机上实际使用的提示词。”

This format works because it is concrete, repeatable, and immediately useful. It feels less like marketing and more like showing your work.

这个格式有效，是因为它具体、可复现、而且立刻可用。它不像营销，更像是在展示你的真实工作过程。
